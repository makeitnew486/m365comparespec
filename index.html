🔵 EA 계약 리스트는 어디서 확인하나요? ▾
고객(구매자) 입장:
• VLSC (microsoft.com/licensing/servicecenter) — EA 계약 상세, 라이선스 수량, True-Up 내역, 만료일 확인. Agreement Admin으로 등록 필요
• Microsoft 365 관리 센터 (admin.microsoft.com) → 청구 → 내 제품에서 현재 구독 확인

영업사원 입장:
• Microsoft Partner Center (partner.microsoft.com) — CSP 파트너인 경우 고객별 구독 현황 조회
• MSX / D365 내부 툴 — Microsoft 직원·공인 파트너는 내부 CRM에서 고객 EA 계약 조회
• 고객사 IT 담당자에게 직접 요청 (Agreement Admin 통해 계약서·현황 공유)

💡 팁: EA 만료일 6개월 전부터 Renewal 대화 시작이 핵심. VLSC에서 만료일 확인 후 선제적으로 연락하세요.
🔵 EA vs CSP, 어떤 계약 방식을 제안해야 하나요? ▾
EA (Enterprise Agreement)
• 대상: 직원 250명 이상 대기업
• 3년 약정, 연간 True-Up 정산
• 장점: 단가 저렴, 통합 관리 편리
• 단점: 약정 기간 중 축소 어려움

CSP (Cloud Solution Provider)
• 대상: 중소기업, 유연성 필요 고객
• 월간/연간 약정 선택 가능
• 장점: 유연한 수량 조정, 파트너 지원
• 단점: EA 대비 단가 다소 높음

💡 판단 기준: 규모가 크고 안정적이면 EA, 빠르게 성장 중이거나 유연성이 필요하면 CSP를 제안하세요.
🔵 NCE 약정 시 주의사항은? ▾
NCE(New Commerce Experience)에서 반드시 고객에게 사전 안내해야 할 내용:

• 연간 약정 선택 시: 약정 기간 내 라이선스 수량 감축 불가. 증가만 가능
• 월간 약정은 연간보다 약 20% 비쌈. 비용 절감이 목적이면 연간 추천
• 약정 갱신일(Anniversary Date) 전후 72시간 내에만 변경 가능
• 중도 해지 시 잔여 기간 요금 환불 불가

💡 팁: 고객이 사용자 수 변동이 잦다면 처음에는 월간으로 시작 후 안정화되면 연간으로 전환을 제안하세요.
🔵 True-Up 정산 시점을 영업에 어떻게 활용하나요? ▾
True-Up은 EA 고객의 연간 라이선스 수량 정산 프로세스로, 영업 최고의 접촉 타이밍입니다.

활용 방법:
• True-Up 1~2개월 전 고객에게 먼저 연락 → "올해 사용자 수 변동 파악 도와드릴게요"
• 사용자 수 증가분 확인 → 추가 라이선스 비용 안내
• 이 시점에 플랜 업그레이드(E3→E5) 또는 Copilot 추가 제안
• 다음 연도 약정 수량 조정 협의

💡 주의: True-Up 미진행 시 라이선스 초과 사용으로 감사(Audit) 위험 있음 — 고객 리스크로 설명하면 설득력 높아집니다.

🔴 "지금 쓰는 게 있어서 바꾸기 어렵다"고 할 때 ▾
현재 사용 환경별 대응:

• Google Workspace 사용 중: 마이그레이션 도구(Google Workspace Migration for Microsoft Exchange) 무료 제공. FastTrack 지원으로 데이터 이전 무료 지원(150석+)
• 온프레미스 Exchange/AD 사용 중: 하이브리드 구성으로 단계적 전환 가능. 당장 전체 이전 안 해도 됨
• Zoom 사용 중: Teams는 Zoom과 연동 가능. 동시에 쓰다가 Teams로 자연스럽게 전환 유도

💡 핵심 메시지: "한 번에 바꾸지 않아도 됩니다. 단계적으로 전환하는 로드맵을 같이 만들어드릴 수 있습니다."
🔴 "보안은 이미 다른 솔루션 쓰고 있다"고 할 때 ▾
중복 투자 해소 관점으로 접근하세요:

• M365 E3/E5에는 Defender, Entra ID, Intune, Purview가 이미 포함
• 기존 보안 솔루션 비용과 M365 포함 보안 기능을 비교 → 중복 계약 정리 기회
• E5의 경우 XDR·SIEM 수준의 보안을 추가 비용 없이 제공

💡 질문 포인트: "현재 보안 솔루션에 월 얼마 쓰고 계세요? M365 안에 이미 포함된 기능과 겹치는 게 있을 수 있습니다."
🔴 "Copilot은 아직 AI 믿기 어렵다"고 할 때 ▾
신뢰도 높이는 포인트:

• Copilot Chat은 무료로 이미 포함 → 먼저 써보게 하세요. 부담 없음
• 기업 데이터는 외부 AI 학습에 사용되지 않음 (엔터프라이즈 데이터 보호)
• AI가 최종 결정하는 게 아니라 초안·요약·분석을 도와주는 도구임을 강조
• 특히 반복 업무(이메일 초안, 회의 요약, 보고서 작성)부터 시작하면 체감 효과 빠름

💡 PoC 제안: "한 팀에서 한 달만 써보고 효과 측정해보시죠. 무료 체험판 연결해드릴게요."
🟢 E3 vs E5 한 문장으로 설명하면? ▾
"E3는 생산성의 기반, E5는 보안의 완성"

• 고급 위협 방지 (피싱·랜섬웨어·제로데이 공격)
• XDR — 이메일·기기·ID·클라우드 앱 전방위 탐지·대응
• Defender for Identity (온프레미스 AD 공격 탐지)
• Power BI Pro (데이터 분석·시각화)
• 고급 eDiscovery·컴플라이언스

💡 E5 추천 고객: 금융·의료·공공기관, 개인정보 대량 처리 기업, 보안 감사 대응이 필요한 곳

• Teams 기능이 필요 없는 부서/직군 (현장 작업자 등)

비용 비교:
E3 Teams 포함: ₩48,600 vs Teams 제외: ₩37,100 → 월 ₩11,500 절감
E5 Teams 포함: ₩77,000 vs Teams 제외: ₩65,500 → 월 ₩11,500 절감

💡 핵심 차이: 무료는 "인터넷 기반 AI 채팅", 유료는 "우리 회사 데이터를 아는 AI 비서"
🟢 F1/F3 Frontline은 언제 제안하나요? ▾
현장 직원(최전방 근무자)이 많은 고객에게 적합:

• 제조·물류·유통 현장 작업자
• 병원·요양원 간호사·간병인
• 유통 매장 직원
• 건설·현장 관리자

F1 vs F3 차이:
• F1: 웹·모바일 앱만 사용 가능. 데스크톱 Office 앱 없음. 가장 저렴
• F3: 데스크톱 Office 앱 포함. E3보다 저렴하면서 기능 충분

💡 혼합 구성 제안: 사무직은 E3, 현장 직원은 F1/F3으로 나눠 라이선스 비용을 최적화하세요.
🟠 FastTrack을 영업에 어떻게 활용하나요? ▾
FastTrack이란: 150석 이상 M365 고객에게 Microsoft가 무료로 제공하는 도입 지원 서비스

제공 내용:
• 배포 계획 수립 지원
• 데이터 마이그레이션 지원 (이메일·파일 등)
• 사용자 교육 자료 제공
• 기술 전문가 직접 지원

영업 활용법:
• "도입이 복잡하지 않냐"는 고객에게 → "Microsoft 전문가가 무료로 도와줍니다"
• 경쟁사 대비 차별점으로 활용
• aka.ms/fasttrack 에서 신청 가능

💡 주의: 150석 미만은 파트너사(CSP)의 지원을 안내하세요.
🟠 무료 평가판(Trial) 전략적으로 활용하는 법 ▾
제공 평가판:
• M365 E3: 1개월 무료
• M365 E3 Teams 제외: 1개월 무료
• M365 Copilot: 일부 고객 대상 무료 체험 가능 (파트너 통해 확인)

Trial 진행 전략:
1. 시작 전 — 체험 목표와 평가 기준을 고객과 미리 합의
2. 2~3주차 — 중간 체크인으로 사용 현황 확인, 문제 해결
3. 만료 1주 전 — 결과 리뷰 미팅 → 구매 결정 유도

💡 핵심: Trial은 자동으로 유료 전환되므로, 만료 전 구매 여부를 반드시 확정하세요. 카드 등록이 필요합니다.
🟠 Renewal(갱신) 타이밍 공략법 ▾
갱신 준비 타임라인:

• 만료 6개월 전: 첫 접촉. 현재 만족도 파악, 사용자 수 변동 확인
• 만료 3개월 전: 플랜 업그레이드 제안, 견적서 제출
• 만료 1개월 전: 계약서 검토 및 서명
• 만료 2주 전: 최종 확인 및 처리

갱신 시 업셀 포인트:
• E3 → E5 업그레이드 (보안 강화 니즈 확인)
• Copilot 라이선스 추가
• 사용자 수 증가에 따른 추가 라이선스

💡 경쟁사 개입 방어: 갱신 6개월 전부터 관계 강화. 경쟁사는 갱신 시점에 집중적으로 공략합니다.
🟠 PoC(개념 증명) 효과적으로 진행하는 방법 ▾
PoC 성공 공식:

1. 범위 한정: 전사 도입 X → 특정 팀(10~30명)으로 시작
2. 측정 지표 합의: "회의 시간 20% 단축", "이메일 처리 시간 30분 절감" 등 구체적 목표 설정
3. 챔피언 지정: 고객사 내 적극적인 담당자를 내부 전도사로 육성
4. 결과 문서화: PoC 종료 후 수치로 결과 정리 → 의사결정권자에게 보고

💡 Copilot PoC 추천 시나리오:
• Teams 회의 자동 요약 → 회의록 작성 시간 절감
• Outlook 이메일 초안 → 응대 시간 절감
• Excel 데이터 분석 → 보고서 작성 시간 절감
🟠 Google Workspace 쓰는 고객 공략법 ▾
핵심 차별점 3가지:

① 보안: Google은 별도 보안 솔루션 필요. M365는 Defender·Purview·Entra ID·Intune 통합 제공
② AI: Microsoft Copilot은 GPT-4 기반으로 모든 Office 앱에 깊게 통합. Google Gemini 대비 업무 데이터 연동 수준이 높음
③ Windows 통합: Windows 11 Enterprise 포함 → 기기 관리까지 일원화

마이그레이션 장벽 해소:
• Google Drive → OneDrive: Microsoft 마이그레이션 도구 무료 제공
• Gmail → Outlook: GWMME(Google Workspace Migration for Microsoft Exchange) 활용
• FastTrack으로 무료 기술 지원(150석+)

💡 공략 타이밍: Google 계약 갱신 6개월 전이 최적. 미리 파악해두세요.

계약·라이선스
EA
Enterprise Agreement
Microsoft와 대기업(250명 이상) 간 3년 볼륨 라이선스 계약. 일괄 구매로 단가가 낮아지며, True-Up(연간 정산)으로 사용자 수를 조정합니다. 엔터프라이즈 영업에서 가장 흔히 접하는 계약 방식입니다.
계약·라이선스
MCA / MPSA / CSP
Microsoft Customer Agreement / Products and Services Agreement / Cloud Solution Provider
MCA: 클라우드 제품 구매를 위한 현대화된 계약. MPSA: 다양한 제품을 하나의 계약으로 구매. CSP: 파트너(리셀러)를 통해 구독하는 방식으로 중소기업에 많이 쓰입니다.
계약·라이선스
True-Up
연간 라이선스 정산
EA 계약에서 연 1회 실제 사용자 수와 계약 수량 차액을 정산하는 프로세스. 사용자가 늘었으면 추가 비용 발생. 고객과 매년 접촉하는 중요한 영업 타이밍입니다.
계약·라이선스
VL (볼륨 라이선스)
Volume Licensing
다수의 라이선스를 일괄 구매해 할인받는 방식. EA, MPSA, Open License 등이 VL의 하위 프로그램입니다. 일정 수량 이상 구매 시 적용됩니다.
계약·라이선스
SA (소프트웨어 어슈어런스)
Software Assurance
라이선스 구매 시 추가하는 유지보수 옵션. 계약 기간 내 새 버전으로 무료 업그레이드, 교육, 기술 지원 혜택 제공. EA에 기본 포함됩니다.
구독·플랜
SKU
Stock Keeping Unit
Microsoft 제품의 개별 라이선스 단위. 예: "M365 E3 SKU", "Copilot SKU". 견적서 작성 시 SKU 코드(예: CFQ7TTC0HNG6)를 기재합니다. 영업 내부에서 플랜을 지칭할 때 자주 씁니다.
구독·플랜
UAL / CAL
User Access License / Client Access License
CAL: 서버 제품(Exchange, SharePoint 등)에 접근하기 위한 클라이언트 라이선스. UAL: 사용자 기준 CAL. M365 구독에 포함되어 별도 구매가 필요 없습니다.
구독·플랜
Add-on (애드온)
추가 라이선스
기본 M365 플랜에 추가 구매하는 라이선스. M365 Copilot, Teams Phone, Audio Conferencing 등이 대표적. 기존 고객에게 업셀(Upsell)할 때 핵심 상품입니다.
구독·플랜
NCE
New Commerce Experience
Microsoft가 2022년부터 도입한 새 CSP 구매 방식. 연간·월간 약정 선택 가능. 월간 약정은 연간보다 약 20% 비쌉니다. 약정 기간 내 라이선스 감축 불가 — 고객에게 반드시 사전 안내가 필요합니다.
구독·플랜
F1 / F3 (Frontline)
Microsoft 365 Frontline Worker 플랜
현장 근무자용 저가형 플랜. F1: 웹·모바일 전용(데스크톱 앱 없음), F3: 데스크톱 앱 포함. 제조·유통·의료 현장 직원이 많은 고객에게 비용 최적화 제안으로 활용하세요.
제품·기술
Entra ID (구 AAD)
Microsoft Entra ID · Azure Active Directory
Microsoft의 클라우드 기반 ID·액세스 관리 서비스. 직원 계정 관리, SSO, MFA 담당. 2023년 Azure AD → Microsoft Entra ID로 명칭 변경. M365 E3/E5에 포함됩니다.
제품·기술
Intune (MDM/MAM)
Microsoft Intune · Mobile Device/Application Management
MDM(기기 관리): 기기 전체에 정책 적용. MAM(앱 관리): 개인 기기에서 업무 앱만 별도 관리. BYOD(개인 기기 업무 활용) 환경에서 필수 솔루션입니다.
제품·기술
Purview / DLP
Microsoft Purview · Data Loss Prevention
데이터 거버넌스·컴플라이언스·정보 보호 통합 플랫폼. DLP는 개인정보·기밀 데이터 외부 유출 차단 정책. 금융·의료·공공기관 고객의 컴플라이언스 핵심 키워드입니다.
제품·기술
XDR / SIEM / Sentinel
Extended Detection & Response / Security Information & Event Management
XDR: 여러 영역 위협을 통합 탐지·대응(E5 포함). SIEM: 보안 이벤트 수집·분석 시스템. Microsoft Sentinel은 Microsoft의 클라우드 SIEM(별도 구매).
제품·기술
Copilot / Agent
Microsoft 365 Copilot · AI 에이전트
Copilot Chat: 무료 포함 기본 AI 채팅. M365 Copilot: 모든 앱 통합 AI (+₩40,500). Agent: 특정 업무를 자율 수행하는 AI 봇. Copilot Studio로 커스텀 에이전트를 직접 제작할 수 있습니다.
IT 기술 용어
MFA / SSO
Multi-Factor Authentication · Single Sign-On
MFA: 비밀번호 외 추가 인증(앱·문자·생체) 요구. 계정 탈취 방지 핵심 수단. SSO: 한 번 로그인으로 여러 앱 자동 접근. 편의성과 보안을 동시에 제공합니다.
IT 기술 용어
Zero Trust
제로 트러스트 보안 모델
"절대 신뢰하지 말고, 항상 검증하라"는 보안 철학. 내부 직원도 매번 인증을 거치며 최소 권한만 부여합니다. Entra ID·Intune·Defender가 Zero Trust 아키텍처를 구현합니다.

하이브리드 / 온프레미스
Hybrid · On-premises

Microsoft 365 테넌트
